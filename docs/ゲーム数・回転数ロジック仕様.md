# ゲーム数・回転数まわりのロジック仕様

## 1. 用語と役割

| 名前 | 役割 |
|------|------|
| **totalRotations** | データランプなど「台の表示している総ゲーム数」に合わせる値。時短・ST中は加算しない（RUSH終了時にまとめて反映する）。 |
| **normalRotations** | **通常ゲームのみ**の累積。回転率・グラフ横軸に使用。時短・ST中は加算しない。 |
| **remainingSupportCount** | 現在の時短/STの残り回数。0で通常に復帰。 |
| **supportPhaseInitialCount** | 現在の時短/ST開始時の「残り回数」の初期値。表示用の「前回当選以降のゲーム数」算出に使用。 |
| **gamesSinceLastWin** | カウントボタンなどに表示する「前回当選からのゲーム数」（時短・ST抜けゲーム含む）。`normalSinceWin + supportPlayed`。 |

## 2. 通常モードでの加算

- **+1 / +10 / +100 タップ時**  
  - 通常時: `totalRotations += n`, `normalRotations += n`  
  - 時短/ST中: `remainingSupportCount` を減らすだけ。`totalRotations`・`normalRotations` は増やさない。  
  - 残りが 0 になったら通常に復帰。

## 3. 時短・STの開始

- 通常当選後: `remainingSupportCount = 時短ゲーム数`, `supportPhaseInitialCount = 時短ゲーム数`  
- RUSH当選後: `remainingSupportCount = ST回数`, `supportPhaseInitialCount = ST回数`  
- いずれも「当選1回」で `totalRotations` は当選時点のまま（時短/ST中は加算しない）。

## 4. RUSH終了時の整合

- **RUSH終了入力**で「獲得出玉」「終了時回転数」「RUSHのゲーム数」を入力。
- **fixTotalChainPrize** で以下を実行:
  1. 最後の RUSH 当選レコードの `prize` を獲得出玉に更新
  2. 最後のレコードの `rotationAtWin` を終了時回転数に更新
  3. 最後のレコードの `rushGamesPlayed` に RUSHのゲーム数を保存
  4. **totalRotations = 終了時回転数** に更新（RUSH中は加算していないため、ここで台表示と合わせる）
- **endRushAndReturnToNormal** で `currentState = .normal`, `remainingSupportCount = 0`, `supportPhaseInitialCount = 0` にし、通常モードに戻す。

これにより、RUSH終了直後は `gamesSinceLastWin = totalRotations - last.rotationAtWin = 0` となり、表示がぶれない。

## 5. 通常モードとSTゲーム数の関係

- **通常モード中**: タップで `totalRotations` と `normalRotations` が同じだけ増える。
- **時短/ST中**: タップでは `remainingSupportCount` だけ減り、`totalRotations`・`normalRotations` は増えない。
- 時短/ST 0 で通常復帰したあと、再度タップすると `totalRotations`・`normalRotations` が増え始める。
- グラフの横軸は `normalRotations`（通常ゲームのみ）なので、時短・ST区間はグラフ上では「進まない」ように見える。

## 6. WinRecord まわり

- **rotationAtWin**: その当選時点の「台表示の総ゲーム数」（RUSH終了時はユーザーが入力した終了時回転数で上書き）。
- **normalRotationsAtWin**: その時点までの通常ゲーム累積（グラフ用）。
- **rushGamesPlayed**: RUSH 1 run で遊んだ総ゲーム数（時短抜け含む）。RUSH終了入力で選択・保存。

## 7. 同期・修正系

- **syncTotalRotations(newTotal:)**: データランプ表示に合わせて `totalRotations` を上書き。通常時は `normalRotations` も整合するよう差分で更新。
- **setInitialDisplayRotation(_:)**: 新規開始時に台表示数に合わせる。`totalRotations = n`, `normalRotations = 0`。
- **correctInitialDisplayRotation(to:)**: 開始時の表示数のみ後から修正。`totalRotations` の差分だけ変更。
