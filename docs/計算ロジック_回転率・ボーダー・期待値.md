# P-stats 計算ロジック整理（表面・実質回転率・ボーダー・期待値）

**主な実装**: `GameLog.swift`（`realRate` / `rotationPer1000Yen` / `effectiveUnitsForBorder` / `totalRealCost`）

---

## 1. 記号

| 記号 | 意味 |
|------|------|
| \(N\) | `normalRotations` — 通常回転のみ |
| \(I\) | `totalInput` — 現金投資累計（pt）、500pt 刻み |
| \(H\) | `holdingsInvestedBalls` — 持ち玉投資の玉数合計 |
| \(B_{500}\) | `interpretedBallsPer500Pt` — 500pt あたり貸玉数 |
| \(r\) | `interpretedPayoutCoefficientPtPerBall` — 払出（pt/玉） |
| 単位 \(U\) | `effectiveUnitsForBorder` — (貸玉換算玉 + \(H\)) / 250 |

**持ち玉1タップの玉数**: `interpretedHoldingsBallsPerTap` — 店の `holdingsBallsPerButton` が正ならその値、**0 のときは \(B_{500}\)**（貸玉と同じ）。

---

## 2. 実費（pt）

\[
\text{totalRealCost} = I + H \cdot r
\]

---

## 3. 表面回転率（`rotationPer1000Yen`）

**定義**: 等価 **250 玉 = 1 単位**の分母で割る。**払出 \(r\) は分母に入れない**。

\[
\text{rotationPer1000Yen} = \frac{N}{U}
,\quad
U = \frac{1}{250}\left(\frac{I}{500}\cdot B_{500} + H\right)
\]

（プロパティ名は歴史的に `rotationPer1000Yen` だが、意味は **回転/等価250玉単位**。）

---

## 4. 実質回転率（`realRate`）

**定義**: 実費を **1000pt = 1 単位**として割る。換金（払出）の影響は主にここ。

\[
\text{realRate} = \frac{N}{\text{totalRealCost}/1000}
= \frac{1000\,N}{I + H r}
\quad (\text{totalRealCost} > 0)
\]

---

## 5. 期待値比

\[
\text{expectationRatio} = \frac{\text{realRate}}{\text{dynamicBorder}}
\quad (\text{dynamicBorder} > 0,\ U > 0)
\]

`dynamicBorder` は店補正後ボーダー（`GameLog.dynamicBorder`）。実質と同じ「経済ベース」で比較。

---

## 6. ゲージの針

概ね **実質回転率 − 実質ボーダー**（`BorderMeterView` / `PlayView.borderForGauge`）。

---

## 7. 旧バージョンとの違い

以前の実装では `realRate` と `rotationPer1000Yen` の式が **逆**だった。保存済み `GameSession.realRotationRateAtSave` は、**新定義では実質（千pt実費）** と一致。表示は可能な限り `totalRealCost` から再計算する（`displayRealRotationRatePer1k`）。
