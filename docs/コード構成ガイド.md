# P-stats コード構成ガイド

開発時の迷子防止用。Xcode は `P-stats` フォルダを **File System Synchronized** で取り込んでいるため、新規 `.swift` は適切なサブフォルダに置けばターゲットに自動参加する。

## ディレクトリの役割

| パス | 内容 |
|------|------|
| **`App/`** | `P_statsApp`（`@main` のみ）、`AppBootstrap`・`MainContentWithContainer`、`HomeView`、設定・履歴・管理・共通ホーム背景など |
| **`Core/`** | SwiftData モデル、`GameLog`、マスタ JSON、永続化ヘルパ、`PlayModeVocabulary`（プレイ画面の用語・`PlayPresentationSurface` と滞在モード表示の一元化） |
| **`Design/`** | `DesignTokens`、`AppGlassStyle`、`AppTypography`、Metal 装飾など UI トークン・共通装飾 |
| **`Extensions/`** | `Int+PtDisplay`、`View+ListSelectionAndKeyboard` など横断 extension |
| **`Features/`** | 画面単位（`Play/` `Analytics/` `MachineShop/` `Common/`） |
| **`Services/`** | API・CloudKit・位置情報・プリセット取得 |
| **`Utilities/`** | 日付フォーマッタ、ハプティクス、検索フィルタなど UI に依存しないユーティリティ |

## 今後の分割候補（`P_statsApp.swift`）

実施済み: `P_statsApp.swift` は **`@main` と `WindowGroup` のみ**。Bootstrap は `App/AppBootstrap.swift`、ロック・スプラッシュ後の本編ラッパーは `App/MainContentWithContainer.swift`。ホーム・設定・履歴・管理は各 `App/*.swift` に分離済み。

分離時は **`internal` のまま**にし、`@Query` や `AppStorage` の引数だけ注意する。

## 重複を増やさないルール

- **和文の日付文字列** … `JapaneseDateFormatters.yearMonthDay` / `yearMonth` を使用（ファイル内に `DateFormatter` を増やさない）
- **グラデ枠・ガラス** … `AppGlassStyle.strokeGradient` と既存のパネル修飾を優先
