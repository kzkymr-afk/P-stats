# 指示書：RUSH 簡易仕様（連チャン回数＋終了時出玉入力）

エージェント・メモ用。実装時は本ドキュメントを参照すること。

---

## 1. 方針

- **通常時**: マスタで「通常大当たり（時短）」「RUSH大当たり（ST回数）」を管理し、機種登録でマスタを選ぶとその内容が反映される。通常時は従来どおり、当たり種別を選んで記録する。
- **RUSH中**: 当たりごとのボーナス種別は記録しない。**連チャン回数だけ**を入力する。RUSH終了時に「RUSH終了」を押し、そのタイミングで**そのRUSHの獲得出玉を1回だけ**入力する。
- **スコープ外**: LT・上乗せ・当選時出玉不明など機種ごとの特殊仕様は、アプリでは扱わない（RUSHは「回数＋終了時出玉」のみ）。

---

## 2. マスタ・機種登録

- **マスタ（Pachinko_Machine_Master 等）**に、次の情報を持たせる（既存の列でよい）:
  - **通常大当たり用**: 時短ゲーム数、当たり種類（出玉・RUSH突入か単発か）など。
  - **RUSH大当たり用**: ST回数（電サポ回数）、必要なら代表的な出玉（表示用のみ。RUSH中の記録には使わない）。
- **機種登録画面**でマスタから機種を選択したとき、上記「通常大当たり（時短）」「RUSH大当たり（ST回数）」がその機種に反映される。
- **RUSH中の「当たり1本ごとのボーナス名・出玉」**はマスタにもアプリにも持たない。

---

## 3. 通常時の動作（変更なし）

- 通常時にヘソで当たり → 大当たり入力で「通常大当たり」か「RUSH大当たり」を選択（マスタ由来の選択肢）。
- **通常大当たり** → 時短に入る。時短終了 or 引き戻し（RUSH当たり）の操作は従来どおり。
- **RUSH大当たり**を選択 → RUSHモードに移行。以降は「RUSH中」の仕様に従う。

---

## 4. RUSH中の動作（新仕様）

### 入力するもの

| 項目 | 内容 |
|------|------|
| **連チャン回数** | RUSHに入ってから、当たるたびに「+1」する（ボタン1つで「1回当たり」とカウント）。ボーナス名・出玉は選ばない。 |
| **獲得出玉** | 入力しない。**RUSH終了時だけ**入力する。 |

### RUSH終了フロー

1. ユーザーが「RUSH終了」ボタンを押す。
2. その時点で**「このRUSHの獲得出玉（玉数）」を入力する**ダイアログ（またはシート）を表示する。
3. 入力を確定したら、そのRUSHを「連チャン回数 N 回・獲得出玉 M 玉」の**1件**として記録し、通常モードに戻る。

### データとして持つもの

- **RUSH 1回あたり**: 連チャン回数（整数）、獲得出玉（整数）。必要なら 開始時回転数・終了時回転数 なども保存してよい。
- RUSH中の「当たり1本目は10R、2本目は15R」のような内訳は**保存しない**。

---

## 5. UI・画面

### RUSH画面（RushFocusView 等）

- **「当たり」ボタン**: 1種類だけ → **「+1回」**（連チャン回数を1増やす）。ボーナス選択は出さない。
- **「RUSH終了」ボタン**: 常に表示。押下 → 出玉入力画面へ。

### RUSH終了時の出玉入力

- 「RUSH終了」押下後、**獲得出玉（玉数）**を入力する画面を表示する。
- **必須入力**とする。未入力のまま確定できないようにする（リマインド可）。
- 確定で保存し、通常画面に戻る。

### 通常時

- 従来どおり「通常大当たり」「RUSH大当たり」の選択と、通常大当たり後の時短・昇格UIを維持する。

### 表示・グラフ

- **通常大当たりモード・RUSHモード**では、上部には収支グラフではなく、**連チャン数を大きく表示**する。
- **収支グラフ**は、ドロワーに表示ボタンを追加。「**現時点での終始**」というボタン。このボタンを押すと、画面上部にポップアップで表示される。

---

## 6. データモデル・保存形式

### RUSH 1回分の記録（例）

| 項目 | 型 | 説明 |
|------|-----|------|
| `rushRunCount` | Int | 連チャン回数 |
| `rushPayout` | Int | 獲得出玉 |
| （任意） |  | `rotationAtStart`, `rotationAtEnd`, `timestamp` など |

- **既存の WinRecord を流用する場合**: RUSH突入は「1件目」として記録し、RUSH中の当たりは「同じRUSH run」に回数だけ加算。RUSH終了時にその run の payout をユーザー入力でセットする、などの形でよい。
- 実装は「**RUSH 1 run = 1 レコード（回数＋出玉）**」が分かればよい。
- **集計**: 「RUSH回数」「RUSH総出玉」「1RUSHあたり平均出玉」などは、上記レコードから算出する。

---

## 7. やらないこと・スコープ外

- RUSH中の当たりごとの**ボーナス名・出玉の選択・記録**は行わない。
- LT（上位RUSH）専用モード・上乗せボーナス・「当たった時点では出玉不明」などの機種特異ロジックは実装しない。
- MachineDetail の「1当たり1行」や、RUSH内の `mode_id` / `next_mode_id` による細かい遷移は、**この仕様では必須としない**（通常時用のマスタがあればよい）。

---

## 8. 完了条件

- [ ] 機種登録でマスタを選択すると、通常大当たり（時短）・RUSH大当たり（ST回数）が反映される。
- [ ] 通常時は従来どおり、通常大当たり／RUSH大当たりを選んで記録できる。
- [ ] RUSH中は「+1回」で連チャン回数だけ増やし、ボーナス選択は行わない。
- [ ] 「RUSH終了」押下時に、獲得出玉を入力する画面が表示され、必須で保存される。
- [ ] 保存データから「RUSH回数」「RUSH獲得出玉」「1RUSHあたり平均出玉」などが集計できる。
- [ ] RUSH終了時の出玉入力を忘れないよう、必須入力 or リマインドがかかっている。
- [ ] 通常大当たりモード・RUSHモードでは、上部には収支グラフではなく、連チャン数を大きく表示する。
- [ ] 収支グラフはドロワーに「現時点での終始」ボタンを追加し、押下で画面上部にポップアップ表示される。
