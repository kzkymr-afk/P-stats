# 機種マスターデータ仕様書（1シート・機種ごとJSON）

## 1. 概要

- **目的**: 1スプレッドシート・1機種1行から `master_out/index.json` と `master_out/machines/{機種ID}.json` を生成する。アプリは **MachineFullMaster** のみデコードする。**`main` ブランチでは `master_out/machines/*.json` を Git 管理しない**（正は **gh-pages** 配信物。ローカル変換時のみフォルダが生成される）。
- **変換**: `scripts/convert_master_one_sheet.py`、GitHub Actions（`deploy-master-out-pages.yml`）、GAS `buildMachineJsonById` が同一ルールでパースする。
- **2026-03 改定**: スプレッドシートは **A〜I の9列のみ**（導入開始日〜ステータス）。**モード列・当たり列・更新対象列（旧 J〜AH）は廃止**。各 `machines/{id}.json` は **`modes: []` / `bonuses: []`** のメタデータのみ（アプリは空配列でデコード可能）。

---

## 2. スプレッドシート

### 2.1 ヘッダー（9列・A〜I）

| 列 | 名前 | 備考 |
|----|------|------|
| A | 導入開始日 | |
| B | 機種ID | 必須・一意 |
| C | 機種名 | |
| D | メーカー | |
| E | 確率 | |
| F | 機種タイプ | |
| G | スペック | |
| H | 特徴タグ | |
| I | ステータス | 出力対象は環境変数 `MASTER_EXPORT_STATUS_VALUES`、既定は「完了」 |

CSV の1行目は上記列名に揃える（**列の順序は不問**。`convert_master_one_sheet.py` が列名で解決する）。

### 2.2 出力対象（ステータス）

`MASTER_EXPORT_STATUS_VALUES`（既定 `完了`、`|` 区切りで複数可）に **含まれる**ステータスの行だけが `machines/{id}.json` として書き出される。機種IDが空・重複・ステータス未対応の行はスキップされる（理由は `export_status.csv` に記録）。

---

## 3. 出力 JSON（MachineFullMaster）

### machines/{id}.json

```json
{
  "machine_id": "...",
  "name": "...",
  "manufacturer": "",
  "probability": "",
  "machine_type": "",
  "intro_start": "",
  "spec": "",
  "tags": "",
  "status": "",
  "modes": [],
  "bonuses": []
}
```

`modes` / `bonuses` は **常に空配列**（後方互換のためキーは残す）。旧JSONにモード・当たりが含まれていても、アプリ側は `decodeIfPresent` 等で読み飛ばし可能。

### index.json

`catalog_rows` 由来の一覧。列名に **「更新対象」** が含まれる旧CSVを読んだ場合のみ `update_target` が付く（9列のみの運用では通常付かない）。

---

## 4. 自動化

- 手元検証用サンプルは **`master_out/master_one_sheet_sample.csv`**（`master_out/README.md` 参照）。
- **GitHub Pages**: `.github/workflows/deploy-master-out-pages.yml` が `convert_master_one_sheet.py` を実行し `gh-pages/master_out/` に配信する。
- **差分更新（機種 JSON）**: 既定では、ワークフローが **前回の `gh-pages` の `master_out/` を取り込んだうえで** 変換する。各 `machines/<id>.json` は **JSON の意味が変わった機種だけ**上書きし、今回のエクスポート対象に含まれない ID の JSON は **削除**する。`index.json` も内容が同じなら書き換えない。全件上書きしたい場合は環境変数 **`MASTER_OUT_FORCE_FULL_WRITE=1`**。
- **手動ワンストップ**: スプレッドシートの GAS メニュー「マスター配信」、または別アプリからの Webhook — **`docs/MASTER_OUT_DEPLOY_GAS.md`**。
- **週次 machines.json**（`.github/workflows/update-master-data.yml`）は別系統（DMM等）。マスタ1シート→`master_out` とは独立。生成物は **`main` にコミット**したうえで、**GitHub Pages（`gh-pages`）のサイトルートの `machines.json` にも同期**され、アプリのデフォルト取得元は Pages URL に統一されている。

---

## 5. アプリ

- `MachineDetailLoader` は **MachineFullMaster** を読み込む。`modes` / `bonuses` が空でも問題ない。
