# 機種マスターデータ仕様書・実装指示書（1シート・機種ごとJSON）

## 1. 概要

- **目的**: 1つのスプレッドシートで機種マスタとモード・当たり詳細を管理し、機種IDごとに1つのJSON（`master_out/machines/{機種ID}.json`）を生成する。アプリは選択した機種のJSONのみを参照する。
- **入力**: 1シート・1機種1行。列を横に並べ、機種マスタ・モード名（0〜7）・当たり（10列・スラッシュ区切り）・ステータスを入力する。
- **出力**:
  - `master_out/index.json` … 機種一覧（machine_id, name）
  - `master_out/machines/{機種ID}.json` … 機種ごとの統合JSON（マスタ＋モード＋当たり）
  - `master_out/export_status.csv` … スキップした行の報告（行番号・機種ID・理由）
- **既存の全機種配列JSON**: 本仕様では廃止し、機種ごとJSON＋index に統一する。
- **他アプリ**: スマホ等でマスタを入力・更新するアプリは、本仕様の「データ辞書」に従い同一フォーマットでデータを扱う。

---

## 2. スプレッドシート構成（現行）

### 2.1 基本ルール

- **1行 = 1機種**。1行目はヘッダー、2行目以降がデータ。
- **機種ID**は必須。空・重複・不正の行はスキップし、報告ファイル（export_status.csv）に記録する。
- **ステータス**列が「スキップ」「無効」「skip」「invalid」等の行はJSON出力しない。

### 2.2 統合ヘッダー（列順・全29列 / A〜AC）

| # | 列名 | 説明 |
|---|------|------|
| 1 | 導入開始日 | 日付文字列（例: 2024/03/01）。 |
| 2 | 機種ID | 必須。一意。ファイル名・一覧のキー。 |
| 3 | 機種名 | 必須。表示用。 |
| 4 | メーカー | メーカー名。 |
| 5 | 確率 | 大当り確率表示（例: 1/49.9）。 |
| 6 | 機種タイプ | st / kakugen 等。 |
| 7 | スペック | ミドル/ライトミドル/甘デジ/その他 等。 |
| 8 | 特徴タグ | ST/遊タイム/c時短/設定付 等（`/` 区切り）。 |
| 9 | ステータス | スキップ・無効等のとき当該行を出力しない。 |
| 10〜17 | モード0 〜 モード7 | 各モードの表示名。電サポ・時短はこのセルに記入。 |
| 18〜29 | 当たり1 〜 当たり12 | 当たりセル（スラッシュ区切り）。詳細は後述。 |

### 2.3 CSV 1行目（ヘッダー）

```
導入開始日,機種ID,機種名,メーカー,確率,機種タイプ,スペック,特徴タグ,ステータス,モード0,モード1,モード2,モード3,モード4,モード5,モード6,モード7,当たり1,当たり2,当たり3,当たり4,当たり5,当たり6,当たり7,当たり8,当たり9,当たり10,当たり11,当たり12
```

### 2.4 当たり列のフォーマット（スラッシュ区切り）

- **新形式（ユニット連結型 / 推奨）**:  
  `当たり名/基本出玉/ユニット出玉/最大連結数/滞在モード番号/移行先モード番号`
  - **ユニット出玉=0** の場合は従来どおりの完結型として扱える
  - 例: `カバネリBONUS/1500/1500/5/1/1`

- **旧形式（完結型 / 後方互換）**:  
  `当たり名/出球数/滞在モード番号/移行先モード番号`
  - 例: `RUSH当り1500/1500/1/1`

- **旧形式（電サポ付き / 後方互換）**:  
  `当たり名/出球数/滞在モード番号/移行先モード番号/電サポ回数`
  - 例: `単発1500/1500/0/0/100`

- 空セル・不正なセルは無視する。滞在モード番号は 0〜7 に対応（モード0〜モード7）。

## 3. 出力JSON

### 3.1 index.json

機種ID・機種名の一覧。スキップされなかった行の機種のみ。

```json
[
  { "machine_id": "12345", "name": "機種名A" },
  { "machine_id": "e-keiji", "name": "e花の慶次" }
]
```

### 3.2 machines/{機種ID}.json（アプリ互換）

既存の `MachineDetail` 形式（machine_id, name, modes）に合わせる。マスタ項目（メーカー・確率・機種タイプ・導入日）は将来の拡張用にルートに追加可能。

```json
{
  "machine_id": "e-keiji",
  "name": "e花の慶次",
  "modes": [
    {
      "mode_id": 0,
      "name": "通常",
      "bonuses": [
        { "name": "単発1500", "payout": 1500, "ratio": 0, "densapo": 100, "next_mode_id": 0 }
      ]
    },
    {
      "mode_id": 1,
      "name": "RUSH",
      "bonuses": [
        { "name": "RUSH当り1500", "payout": 1500, "ratio": 0, "densapo": 10, "next_mode_id": 1 }
      ]
    }
  ]
}
```

- `ratio`: スプレッドシートに振り分け率が無い場合は 0。
- `densapo`: 当たりセルに5番目があればそれ、なければ 0。

### 3.3 export_status.csv（スキップ報告）

スキップした行を記録。ヘッダー例: `行番号,機種ID,理由`

```
行番号,機種ID,理由
2,,機種IDが空
5,12345,機種ID重複
7,99999,ステータスによりスキップ
```

## 4. データ辞書（他アプリ連携用）

| 項目 | キー／列名 | 型 | 必須 | 備考 |
|------|------------|-----|------|------|
| 機種ID | 機種ID / machine_id | 文字列 | ○ | 一意。ファイル名に使うため英数字・ハイフン推奨。 |
| 機種名 | 機種名 / name | 文字列 | ○ | 表示用。 |
| メーカー | メーカー / manufacturer | 文字列 | - | |
| 確率 | 確率 / probability | 文字列 | - | 例: 1/49.9 |
| 機種タイプ | 機種タイプ / machine_type | 文字列 | - | st, kakugen 等 |
| 導入日 | 導入日 / introduction_date | 文字列 | - | 例: 2024-03-01 |
| モード名 | モード0〜モード7 | 文字列 | - | mode_id 0〜7 の表示名 |
| 当たり | 当たり1〜当たり10 | 文字列 | - | 「名/出球/滞在mode/移行先mode[/電サポ]」 |
| ステータス | ステータス / status | 文字列 | - | スキップ・無効等のとき出力対象外 |

---

## 5. 変換スクリプト

- **スクリプト**: `scripts/convert_master_one_sheet.py`
- **入力**: 上記25列のCSV（ローカルパスまたは環境変数 `MASTER_ONE_SHEET_CSV_URL` でURL指定）
- **出力**: `master_out/index.json`, `master_out/machines/{機種ID}.json`, `master_out/export_status.csv`

実行例:

```bash
python scripts/convert_master_one_sheet.py master_out/master_one_sheet.csv
MASTER_ONE_SHEET_CSV_URL="https://docs.google.com/.../export?format=csv&gid=0" python scripts/convert_master_one_sheet.py
```
