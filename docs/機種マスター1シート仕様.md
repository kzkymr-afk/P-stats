# 機種マスターデータ仕様書（1シート・機種ごとJSON）

## 1. 概要

- **目的**: 1スプレッドシート・1機種1行から `master_out/index.json` と `master_out/machines/{機種ID}.json` を生成する。アプリは **MachineFullMaster** のみデコードする。
- **変換**: `scripts/convert_master_one_sheet.py`、GitHub Actions（`deploy-master-out-pages.yml`）、GAS `buildMachineJsonById` が同一ルールでパースする。

---

## 2. スプレッドシート

### 2.1 ヘッダー（29列・A〜AC）

| 列 | 名前 |
|----|------|
| A | 導入開始日 |
| B | 機種ID（必須・一意） |
| C | 機種名 |
| D〜H | メーカー、確率、機種タイプ、スペック、特徴タグ |
| I | ステータス（出力対象は環境変数 `MASTER_EXPORT_STATUS_VALUES`、既定「完了」） |
| J〜Q | **モード0** 〜 **モード7** |
| R〜AC | **当たり1** 〜 **当たり12** |

### 2.2 モード列（必須形式）

各セルは **スラッシュ3項目以上**:

`mode_N / 表示名 / densapo [/ ui_role]`

- **N は 0〜7 のみ**  
  - **`mode_0` = 通常時**（必ず定義すること。当たり行がある場合は `mode_0` 行が無いと JSON 化エラー）  
  - **`mode_1`〜`mode_7` = 特殊モード**
- `densapo`: 数値、`INF`、`∞`
- `ui_role`（省略可）: `n`=通常系(0)、`r`=RUSH系(1)、`l`=LT(2)  
  - 省略時: **mode_0 → `n`（0）**、**mode_1〜7 → `r`（1）**

### 2.3 当たり列（9項目・スラッシュ区切り・厳密）

```
あたりID / あたり名 / 基本出玉 / ユニット出玉 / 最大連結数 /
滞在モードID / 移行先モードID / 分岐ラベル / 昇格先あたりID
```

- **あたりID**: マスタ内一意（JSONの `bonus_id`）。`bonus_1` 形式推奨。数値のみなら `bonus_{数}` に正規化。
- **あたり名**: 表示用
- **滞在・移行**: `mode_0`〜`mode_7` または整数 **0〜7**
- **分岐ラベル**: 同一滞在モードで同名当たりを分けるときに使用
- **昇格先**: 無しは `-` または空。値は **当たりID** と一致する必要あり（`promotion_id` → 参照先の `bonus_id`）。変換時に同じ行内の `bonus_id` 集合へ存在チェックあり。

空の当たりセルはスキップ。項目数が9でないセルはエラー。

## 3. 出力 JSON（MachineFullMaster）

### machines/{id}.json

```json
{
  "machine_id": "...",
  "name": "...",
  "manufacturer": "",
  "probability": "",
  "machine_type": "",
  "intro_start": "",
  "spec": "",
  "tags": "",
  "modes": [
    { "mode_id": 0, "name": "通常", "densapo": 100, "ui_role": 0 }
  ],
  "bonuses": [
    {
      "bonus_id": "bonus_1",
      "name": "単発",
      "base_payout": 1500,
      "unit_payout": 0,
      "max_concat": 1,
      "stay_mode_id": 0,
      "next_mode_id": 0,
      "branch_label": "",
      "promotion_id": null,
      "next_ui_role": 0
    }
  ]
}
```

`next_ui_role` は変換スクリプトが **移行先モード** の `ui_role` をコピーして付与する。

---

## 4. 自動化

- **GitHub Pages**: `.github/workflows/deploy-master-out-pages.yml` が `convert_master_one_sheet.py` を実行し `gh-pages/master_out/` に配信する。
- **週次 machines.json**（`.github/workflows/update-master-data.yml`）は別系統（DMM等）。マスタ1シート→`master_out` とは独立。

---

## 5. アプリ

- `MachineDetailLoader` は **MachineFullMaster のみ**読み込む。
- `bonus_id` は必須文字列。昇格は `promotion_id` と **同じ文字列の `bonus_id`** を参照する。
