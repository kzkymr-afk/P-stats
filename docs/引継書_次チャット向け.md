## 2. 機種マスター（スプレッドシート → JSON → アプリ）

### 2.1 単一の正データ形式: `MachineFullMaster`

- **JSON**: `master_out/machines/{機種ID}.json` + `master_out/index.json`
- **アプリ**: `MachineDetailLoader` は **`MachineFullMaster` のみ**デコード（ネスト旧形式 `MachineDetail` は削除済み）

### 2.2 スプレッドシート仕様（固定・後方互換なし）

詳細は **`docs/機種マスター1シート仕様.md`**。

| 項目 | 内容 |
|------|------|
| **モード列**（モード0〜7） | `mode_N/表示名/densapo[/ui_role]`。**N は 0〜7**。`mode_0`=通常、`mode_1`〜7=特殊。`ui_role`: n/r/l（省略時: mode_0→通常系、他→RUSH系扱い） |
| **当たり列**（当たり1〜12） | **スラッシュ9項目固定**: `あたりID/あたり名/基本出玉/ユニット出玉/最大連結数/滞在モードID/移行先モードID/分岐ラベル/昇格先あたりID` |
| **bonus_id** | JSON 必須。昇格は `promotion_id` が別行の **`bonus_id` と一致**すること |

| 経路 | ファイル / 設定 |

### 2.4 主要 Swift モデル

- `P-stats/Core/MachineDetailLoader.swift` … 取得・バンドル `master_out`

## 3. 遊技状態・UI ロール

- **`GameLog.currentModeID`**: マスタ上の **mode_id（0〜7）**（滞在モード表示・旧データ互換）
- **`GameLog.currentModeUiRole`**: **0=通常系 / 1=RUSH系 / 2=LT**（表示・履歴の `WinType` と整合）
- **実戦の当たり記録**は **`大当たりモード`**（`isBigHitMode` / `BigHitModeView`）で連チャン回数を扱い、確定時に `commitBigHitSessionToNormal` で `winRecords` に1件追加する流れが主経路

続きから復元: `ResumableState` に `currentModeUiRole`・大当たりモード用フラグあり（`ResumableStateStore`）。

## 4. 最近入れた UX / 永続化まわり

- **起動直後のタップ**: ホーム `mainContent` の **初期 `opacity(0)` を廃止**（オフセットのみアニメ）。`AppBootstrapView` でコンテナ準備後 `Task.yield()`。`LaunchView` 終了時も `yield` 後に `onFinish`
- **オートセーブ**: `ResumableStateStore.autosave(from:force:)`
  - **`inactive` / `background`**: `force: true`（ホーム・実戦両方）
  - **実戦中**: 約 **75秒** タイマー + **最短25秒**の非強制保存スロットル（`PlayView`）
  - セッション確定は従来どおり **`save(from:)`**（スロットルなし）

## 7. clasp 再設定（次チャットで対応推奨）

### 「Project settings not found」の原因と対処

**原因（典型）**

- **`gas_project/.clasp.json` がない** … リポジトリでは `.clasp.json` を **`.gitignore`** しているため、clone しただけでは作られず、別 PC やクリーン環境では消えている
- **`scriptId` が誤り** … スプレッドシート ID と取り違え、または別プロジェクトの ID を書いている
- **作業ディレクトリが違う** … `clasp push` を `gas_project` 以外で実行している

**対処の要点**

正しい **スクリプト ID**（下記取得方法）で `.clasp.json` を用意し、`gas_project` で `clasp push` する。手順の詳細は **`gas_project/README.md`**。

### 再設定の 2 通り（`clasp clone` / `.clasp.json.example`）

| 方法 | 手順の要約 | 注意 |
|------|------------|------|
| **案 A: `clasp clone`** | `cd gas_project && clasp clone <SCRIPT_ID>` … `.clasp.json` が自動生成される | **clone 直後、ローカル `gas.gs` が GAS エディタの内容で上書きされる**ことがある。リポジトリ版が新しければ `gas.gs` を戻してから `clasp push` |
| **案 B: `.clasp.json.example`** | `cp .clasp.json.example .clasp.json` のあと **`scriptId` のみ** GAS のスクリプト ID に書き換え | **`clasp clone` を使わず**に紐づけできる。テンプレはリポジトリにコミット済み（`gas_project/.clasp.json.example`） |

どちらでも最後に `clasp push` で反映確認。

### 現状（GAS / clasp）

- **GAS**: Script Properties（`GITHUB_PAT`, `GITHUB_REPO`）設定済み。GitHub 側の Variables に `MASTER_ONE_SHEET_CSV_URL` 等の設定済み
- **clasp**: 環境ごとに `.clasp.json` が必要。上記エラー時は本節の再設定。**clasp が使えない間**は `gas.gs` を GAS エディタへ**手動コピペ**でも同じ反映が可能

## 10. 分析 UI・履歴編集（2026-03-28 追記）

本節は「分析画面の再構成」「履歴詳細」「履歴編集の区間入力」「表記統一」の**実装済み仕様**と、触るファイルの索引。

### 10.1 用語・表記

- UI・コメントで **「実践」は「実戦」** に統一（誤記扱い）。例: 実戦履歴、実戦回転率。

### 10.2 分析ダッシュボード（`AnalyticsDashboardView.swift`）

- **全般**: 上部固定の通算サマリー撤廃済み。スクロール内に **通算サマリーカード**（タップで遷移なし）。最下部付近に **「実戦履歴」** ボタン → 全体の遊技履歴一覧へ。
- **通算サマリー行構成**（1行あたり最大2項目、順序は仕様どおり）: タイトル「通算サマリー」→ 通算遊技回数 / 通算成績（1行目は **左: 通算成績、右: 通算遊技回数**）→ 総投資・総回収 → 勝利・敗北 → 勝率・平均初当たり確率 → 通算期待値・期待値との差 → 平均回転率・公式ボーダーとの差 → 1日最大投資・最高回収 → 最高収支・平均収支。
- **パネル見出し**: 収支トレンド・ヒートマップ等の **パネル内タイトル** は通算サマリーと同系の **`AppTypography.panelHeading`** で統一。
- **店舗・機種・メーカー一覧**: `AnalyticsGroupCard` の **コンパクト4行**（店名/機種名/メーカー名のみ1行目、以下は通算成績・通算遊技回数、勝率・平均初当たり確率、平均回転率・公式ボーダーとの差）。`compactListSessions` と `machinesByNameForCompact` でスコープセッションを渡す。
- **区切り線**: `OverviewTotalSummaryPanel` およびコンパクトカード行の間に **薄い横線**（`Color.white.opacity(0.12)` 程度）。
- **個別店舗 / 個別機種 / 個別メーカー（子階層）**: 最上段に **`OverviewTotalSummaryPanel`**（そのスコープのセッションのみ）。続けて **曜日傾向**・**特定日傾向**（グラフのサブタイトルで「欠損・余剰」は **「期待値との差（欠損・余剰）」**）。
- **棒グラフ**: `WeekdayTendencySection` / `SpecificDayBarChartSection` の `BarMark` に **`width: .ratio(0.8)`**。
- **履歴詳細（孫）** `AnalyticsSessionDetailView`: **List 廃止** → パネル＋`ScrollView`。先頭は **「実戦日」**（旧「記録日時」）。**下 padding 約 180pt** でドックに隠れないようにスクロール可能。右上「編集」で `GameSessionEditView`。

### 10.3 集計ロジック

- **`AnalyticsOverviewTotalSummary`**（`AnalyticsEngine.swift` ほか）: 通算サマリー・子画面のパネルで共有。
- **`AnalyticsGroup`**: グループ行用。コメント・プロパティの説明は「実戦回転率」表記に揃え済み。

### 10.4 履歴編集（`GameSessionEditView.swift`）

- **詳細編集**は **スクロール＋ガラス風カード**。**左ラベル・右入力**。
- **セッション内訳（ブロック）**（`+` で追加、複数時は `−` で削除）:
  - 初当たりまでの回転数
  - 初当たりまでの投資（pt）／持ち玉投資（玉）
  - この区間の大当たり回数
  - この区間の回収出玉
- **総回転（保存）**: 空欄なら各区間の回転の合計。**保存する総回転 ≠ 区間合計**のとき、現金・持ち玉・`totalRealCost` を **回転比でスケール**（加重ではなく一様スケールで誤差吸収）。
- **最終の総回収出玉**: `totalHoldings`。**RUSH 当選**は各区間の大当たりの **合算**。通常／LT は別欄。
- **永続化**: `GameSession.editSessionPhasesJSON` にブロック配列を JSON 保存。未設定の旧データは **1 ブロック**に読み替え。

### 10.5 モデル（`Models.swift`）

- `GameSession.editSessionPhasesJSON: String`（デフォルト `""`）。

### 10.6 型・ヘルパー（`GameSessionEditPhases.swift`）

- `GameSessionEditPhaseStored`（`id`, `rotationsUntilFirstHit`, `investmentCashPt`, `investmentHoldingsBalls`, `bigHitCount`, `recoveryHoldingsBalls`）。
- `GameSessionEditPhasesStorage`: `decode` / `encode` / `rawTotalRealCost` / `sumRotations` など。

### 10.7 併せて差分が出ているファイル（ホーム・履歴一覧）

- `HomeView.swift`, `HomeInsightMetrics.swift`, `HomeIntegratedInfoPanel.swift`, `HistoryAndManagementViews.swift`: **実戦**表記など。
- `GameLog.swift`: コメントの「実戦」表記。

### 10.8 ビルド・プロジェクト

- `P-stats` フォルダは **File System Synchronized Root** のため、`GameSessionEditPhases.swift` を `P-stats/Core/` に置けばターゲットに含まれる。
