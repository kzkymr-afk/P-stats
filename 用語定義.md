# アプリ画面・用語定義一覧（統計シミュレーション用）

開発・チャットにおける認識の齟齬を防ぐための画面名と用語の定義です。Apple審査ガイドライン5.3対応のため、表示は「pt」「成績」「理論値」「当選」「基準値」「払出係数」「投入」を用います。

## 1. アプリの主要な画面（ページ）とUI

| 画面・UI名 | 英語名（View名） | 役割・定義 |
| :--- | :--- | :--- |
| **通常モード（実戦）** | `PlayView` | 回転数・投入（現金・持ち玉）を入力し、状態をトラッキングするメイン画面。下部から**大当たりモード**へ遷移する。 |
| **大当たりモード** | `BigHitModeView` | 実戦中に連チャン回数などを扱うオーバーレイ。`PlayView` から開く。 |
| **省エネモード** | `PowerSavingModeView` | スワイプ等のジェスチャーで、画面を見ずに投入や回転数をカウントする。下部から大当たりモードへ遷移可能。 |
| **ドロワー / パネル** | `InsightPanelView` | 画面下部から引き出して表示（または常時表示）する情報パネル。「実際の成績」「理論値収支」「当選回数修正」などを含む。 |
| **設定画面** | `SettingsTabView` | ホーム画面下部の「Settings」タブからアクセス。テーマ変更、アプリロック、遊技のデフォルト設定や背景画像など。 |
| **機種・店舗選択画面** | `MachineShopSelectionView` | 遊技開始前に、打つ機種やプレイする店舗を選択する。 |
| **機種編集画面** | `MachineEditView` | 機種のスペック（確率、基準値、ラウンド構成、賞球数、突入率・継続率など）を登録・編集する。 |
| **店舗編集画面** | `ShopEditView` | 店舗の貸玉料金（1000ptあたりの玉数）や払出係数、特定日などを登録・編集する。 |
| **当たり種類一覧** | `PrizeSetListView` | 「10R 1500玉」などの出玉構成（ライブラリ）を管理する。 |
| **分析画面** | `AnalyticsDashboardView` | 過去のプレイ履歴（GameSession）を基に、成績や理論値、勝率などを分析・振り返るダッシュボード。 |

※ 画面上部に出す「**滞在モード**」表示（マスタの `modes[].name` 優先、なければ 通常/RUSH/LT/時短/電サポ のフォールバック）は `GameLog.stayModeDisplayName(machineMaster:)`（`PlayRuntimeModeVocabulary`）で一元化。

## 2. 遊技記録・機能に関する用語（UI上の表示名など）

| 用語 | 役割・定義 |
| :--- | :--- |
| **投入（現金）** | 現金を使った投入記録。 |
| **投入（持ち玉）** | 獲得した出玉（持ち玉）を利用した投入。※設定により125玉未満でも投入として記録・計算される仕様あり。 |
| **実際の成績（今回の成績）** | 最終的な「回収額（持ち玉×払出係数） − 投入額」で算出される実損益（pt）。 |
| **理論値収支（理論上の損益）** | 公式基準値や回転率、払出係数などを基に算出した、「理論上いくら勝っている/負けているか」の数値（pt）。 |
| **欠損・余剰** | 「実際の成績 − 理論値収支」の差額。プラスなら余剰（理論よりツイている）、マイナスなら欠損。 |

## 3. スペック・内部計算に関する専門用語

| 用語 | コード上の定義 | 役割・定義 |
| :--- | :--- | :--- |
| **通常当選** | `WinType.normal` | 履歴・編集用の区分。RUSH/LT 系と区別する単発当選。 |
| **RUSH (確変/ST)** | `WinType.rush` | 履歴・編集用の区分。電サポが継続しやすい状態での当選。 |
| **総回転数** | `normalRotations` | 通常時（電サポ中・当選中を除く）に回した累計回転数。 |
| **電サポ / 時短ゲーム数** | `supportLimit` / `timeShortRotations` | 当選後やRUSH中の、玉を減らさずに回せる状態。STの規定回数や、通常当選後の時短回数など。 |
| **払い出し（出玉）** | `payoutBalls` / `defaultPrize` | 当たった際に盤面から払い出される玉数（公表値。例: 1500玉）。 |
| **1R純増（純増出玉）** | `netPerRound` / `netBalls` | 「払い出し玉数 − 打ち出し玉数（ラウンド数×カウント賞球数）」で計算される、**実際に手元に残る玉数**。基準値や理論値計算の根拠となる重要指標。 |
| **カウント数（賞球数）** | `countPerRound` | 1ラウンドあたりにアタッカーへ入れる規定玉数（打ち出し玉数の計算に使用。例: 10カウント）。 |
| **突入率 / 継続率** | `entryRate` / `continuationRate` | RUSHへの突入確率（%）、およびRUSHがループする確率（%）。（公式基準値が未入力時の自動計算に使用される） |
| **払出係数** | `payoutCoefficient` | 1玉あたりの換金単価（pt/玉。例: 等価なら 4.0 pt/玉）。 |
| **貸玉料金** | `ballsPerCashUnit` | 現金投入の際のレート（例: 1000ptあたり 125玉など）。 |
