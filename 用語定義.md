# アプリ画面・用語定義一覧

開発・チャットにおける認識の齟齬を防ぐための画面名とパチンコ用語の定義です。

## 1. アプリの主要な画面（ページ）とUI

| 画面・UI名 | 英語名（View名） | 役割・定義 |
| :--- | :--- | :--- |
| **通常モード** | `PlayView` | 通常時の回転数や投資（現金・持ち玉）を入力し、状態をトラッキングするメイン画面。 |
| **省エネモード** | `GesturePlayView` | スワイプ等のジェスチャー操作を利用して、画面を見ずに投資や回転数をカウントする機能。 |
| **収支グラフ** | `ChanceModeView` / `RushFocusView` | 大当たり中などに表示される、総回転数を横軸、収支を縦軸とした折れ線グラフ。 |
| **RUSH特化画面** | `RushFocusView` | RUSH（確変/ST）突入時に表示される専用画面。収支グラフをRUSH用のパネル装飾で表示する。 |
| **ドロワー / パネル** | `InsightPanelView` | 画面下部から引き出して表示（または常時表示）する情報パネル。「実際の収支」「期待値収支」「大当たり回数修正」などを含む。 |
| **設定画面** | `SettingsTabView` | ホーム画面下部の「Settings」タブからアクセスする画面。テーマ変更、アプリロック、遊技のデフォルト設定や背景画像の設定などを行う。 |
| **機種・店舗選択画面** | `MachineShopSelectionView` | 遊技開始前に、打つ機種やプレイする店舗を選択する画面。 |
| **機種編集画面** | `MachineEditView` | 機種のスペック（確率、ボーダー、ラウンド構成、賞球数、突入率・継続率など）を細かく登録・編集する画面。 |
| **店舗編集画面** | `ShopEditView` | 店舗の貸玉料金（1000円あたりの玉数）や交換率、特定日などを登録・編集する画面。 |
| **当たり種類一覧** | `PrizeSetListView` | 「10R 1500玉」などの出玉構成（ライブラリ）を管理する画面。 |
| **分析画面** | `AnalyticsDashboardView` | 過去のプレイ履歴（GameSession）を基に、収支や期待値、勝率などを分析・振り返るダッシュボード。 |

## 2. 遊技記録・機能に関する用語（UI上の表示名など）

| 用語 | 役割・定義 |
| :--- | :--- |
| **現金投資** | 現金（サンド）を使った投資記録。 |
| **持ち玉投資** | 獲得した出玉（持ち玉）を利用した投資。※設定により125玉未満でも投資として記録・計算される仕様あり。 |
| **実際の収支（今回の収支）** | 最終的な「回収額（持ち玉×交換率） − 現金投資額」で算出される実損益（円）。 |
| **期待値収支（理論上の損益）** | 公式ボーダーや回転率、交換率などを基に算出した、「理論上いくら勝っている/負けているか」の数値（円）。 |
| **欠損・余剰** | 「実際の収支 − 期待値収支」の差額。プラスなら余剰（理論よりツイている）、マイナスなら欠損。 |

## 3. パチンコスペック・内部計算に関する専門用語

| 用語 | コード上の定義 | 役割・定義 |
| :--- | :--- | :--- |
| **通常大当たり** | `WinType.normal` | RUSH（確変・ST）に突入しない大当たり。大当たり後は通常状態に戻る（または時短へ）。 |
| **RUSH (確変/ST)** | `WinType.rush` | 電サポ（右打ち）が継続して連チャンしやすい状態での大当たり。アプリ上は確変タイプ（手動終了）とSTタイプ（規定回数で自動終了）を区別。 |
| **総回転数** | `normalRotations` | 通常時（電サポ中・大当たり中を除く）に回した累計回転数。 |
| **電サポ / 時短ゲーム数** | `supportLimit` / `timeShortRotations` | 大当たり後やRUSH中の、玉を減らさずに回せる状態。STの規定回数や、通常大当たり後の時短回数など。 |
| **払い出し（出玉）** | `payoutBalls` / `defaultPrize` | 当たった際に盤面から払い出される玉数（公表値。例: 1500玉）。 |
| **1R純増（純増出玉）** | `netPerRound` / `netBalls` | 「払い出し玉数 − 打ち出し玉数（ラウンド数×カウント賞球数）」で計算される、**実際に手元に残る玉数**。ボーダーや期待値計算の根拠となる重要指標。 |
| **カウント数（賞球数）** | `countPerRound` | 1ラウンドあたりにアタッカーへ入れる規定玉数（打ち出し玉数の計算に使用。例: 10カウント）。 |
| **突入率 / 継続率** | `entryRate` / `continuationRate` | RUSHへの突入確率（%）、およびRUSHがループする確率（%）。（公式ボーダーが未入力時の自動計算に使用される） |
| **交換率** | `exchangeRate` | 1玉あたりの換金単価（例: 等価なら 4.0 円/玉）。 |
| **貸玉料金** | `ballsPerCashUnit` | 現金投資の際のレート（例: 1000円あたり 125玉など）。 |
