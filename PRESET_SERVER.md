# プリセット機種をサーバーで管理するには

管理人はアプリではなく **サーバー（または管理画面）** でプリセットを追加・編集・削除します。アプリはそのサーバーのAPIを呼んでプリセット一覧を表示します。

---

## 1. サーバー側で用意するもの

### 1.1 API の形

- **GET** でプリセット一覧を返す API を用意する。
- 例: `https://your-server.com/api/presets`
- レスポンスは **JSON の配列**（下記の形）。

### 1.2 JSON の例

```json
[
  {
    "name": "機種名",
    "machineTypeRaw": "st",
    "supportLimit": 160,
    "defaultPrize": 1500,
    "probability": "1/319.5",
    "border": "16.0",
    "prizeEntries": [
      { "label": "10R（1500玉）", "rounds": 10, "balls": 1500 }
    ]
  }
]
```

- `machineTypeRaw`: `"st"` または `"kakugen"`
- `prizeEntries`: 当たり種類の配列（0個でも可）

### 1.3 管理のしかた

- サーバー上の **管理画面** や **DB を直接編集** して、この JSON を返すようにする。
- 例: Vercel / Firebase / 自前サーバーで JSON を返す API を1本用意し、そこにあなただけがアクセスしてデータを入れる。

---

## 2. アプリ側でやること

- **プリセット一覧の URL** を `PresetServiceConfig.swift` の `presetListURL` に設定する。
  - 空文字のときは、アプリ内（SwiftData）のプリセットがあればそれを使う（通常はサーバー運用なら空でよい）。
  - 例: `static var presetListURL: String { "https://your-server.com/api/presets" }`
- 「プリセットから選ぶ」を開くと、その URL に **GET** でアクセスし、返ってきた JSON を一覧表示する。
- 表示は **上位30件**（検索時は該当するものすべて）。

---

## 3. 運用の流れ

| 役割 | やること |
|------|----------|
| **管理人（あなた）** | サーバー／管理画面でプリセットを追加・編集・削除。アプリは触らない。 |
| **一般ユーザー** | アプリの「プリセットから選ぶ」で、サーバーから取得した一覧を見て採用する。 |

アプリ内の「プリセット管理画面」は使わず、データの正はサーバー1本にすると、管理が楽になります。
